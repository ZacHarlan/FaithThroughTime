## Session 1 — Initial Build

**Goal:** Build a production-ready interactive Bible timeline web application.

**Completed:**
- .NET 9 solution with 3 projects: BibleTimeline.Web, BibleTimeline.Tests, BibleTimeline.E2E
- SQLite schema: 10 tables + 2 FTS5 virtual tables + auto-sync triggers
- Seed data: 47 people, 52 events, 66 books, 15 locations with junction tables
- Dapper data access layer (BibleTimelineDb.cs) with MatchNamesWithUnderscores
- Minimal API endpoints: timeline, periods, books, filters, person detail, event detail, search
- Vanilla JS frontend: 7 IIFE modules (api, state, timeline, filters, search, detail-panel, app)
- D3.js SVG timeline with zoom (0.1x–200x), pan, period bands, confidence display
- Dark-themed CSS with responsive layout
- 16 unit tests (in-memory SQLite) + 23 integration tests (WebApplicationFactory) — all passing
- 9 Playwright E2E test scenarios
- ARCHITECTURE.md documentation

**Bugs Fixed:**
- Dapper column mapping: Added DefaultTypeMap.MatchNamesWithUnderscores = true
- SQL file loading: Switched from file paths to embedded assembly resources
- Seed data FK mismatches: Corrected junction table foreign key IDs
- Stale DB: Renamed old DB to force re-seed
- NuGet version: Pinned Microsoft.AspNetCore.Mvc.Testing to 9.0.4 (not 10.0.5)
